## 1. Drag & Drop Visual Feedback

### What Changed
When you drag components onto the canvas, you'll now see:
- **Highlighted Drop Zones**: Areas that accept the component light up in blue
- **Drag Preview**: A preview of what you're dragging
- **Success Notification**: Confirmation message when you drop an item

### How to Use
1. Click and hold a component from the left panel
2. Watch as valid drop zones highlight
3. Move your mouse over the canvas
4. Release to drop the component
5. See the success message appear

### Tips
- Green checkmark ‚úì = Drop was successful
- Keep an eye on the blue highlights to see where you can drop
- Works with all component types

## 2. Template History & Recovery

### What Changed
Your template now automatically saves versions as you work, allowing you to:
- **Recover Old Versions**: Go back to any previous state
- **See File Sizes**: Understand how complex each version is
- **Track Changes**: Know exactly when changes were made

### How to Use

#### Accessing History
1. Look for the history icon in the toolbar (or use panel visibility settings)
2. Click to open the Template History panel
3. See all your previous versions listed

#### Restoring a Version
1. Open the history panel
2. Click on any previous version
3. Your template instantly reverts to that state
4. Keep the changes by continuing to work, or try another version

#### Understanding the Panel
- **Time**: When the version was saved (e.g., 2:45:30 PM)
- **File Size**: How large the template is (in KB)
- **Active Version**: The current version is highlighted in blue

### Tips
- Versions auto-save after every change (you don't need to do anything)
- Up to 20 versions are kept automatically
- Oldest versions are removed when you exceed the limit
- All version data is lost when you close the editor (it's session-based)

## 3. Inline Context Tooltips

### What Changed
Help is now built right into the interface. Almost every button now has a tooltip that explains what it does.

### How to Use

#### With Mouse
1. Hover your mouse over any button with a **?** icon
2. A tooltip appears showing what the button does
3. Move away, and it disappears

#### With Keyboard
1. Use Tab to navigate to any button
2. Focus (highlighted) buttons show their tooltip
3. Press Tab again to move to the next control

### Common Tooltips You'll See
- **Save**: "Save template to Anki"
- **Export**: "Export template as HTML"
- **Preview**: "Preview card appearance"
- **Undo/Redo**: "Undo/Redo last change (Ctrl+Z)"
- **Device Buttons**: "View on [Device Name]"
- **Panel Buttons**: Description of what each panel does

### Tips
- Tooltips work in both light and dark modes
- They're accessible with both mouse and keyboard
- All tooltips are short and to-the-point
- No need to click‚Äîjust hover or focus

## 4. Customizable Interface Layout

### What Changed
You can now customize which panels and buttons appear, and adjust the interface to suit your workflow.

### How to Customize

#### Access Settings
1. Look for the **‚öôÔ∏è (settings)** button in the bottom-right corner
2. Click to open the Customization Panel
3. A panel slides in from the left showing all options

#### Panel Visibility
Under "Panel Visibility" section:
- Check/uncheck each panel you want to show or hide
- Example: Uncheck "Blocks Panel" to hide the component list
- Changes apply immediately

#### Toolbar Buttons
Under "Toolbar Buttons" section:
- Check/uncheck buttons you want to show or hide
- Example: Hide "Validate" if you don't use it
- Keep the buttons you use most

#### Layout Options
Under "Layout Options" section:
- **Compact Mode**: Check to make everything more compact
- **Right Panel Width**: Adjust the width of the right panel (150-500 pixels)
  - Smaller = more canvas space
  - Larger = more room for properties

#### Save Your Settings
1. Click "Save Changes" to apply and remember your settings
2. Your settings are saved automatically
3. Next time you open the editor, your layout is restored

#### Reset to Defaults
1. Click "Reset to Defaults" if you want to start over
2. Confirm when asked
3. All panels and buttons return to default visibility

### Customization Examples

**For Minimalists**
- Hide: Layers, History
- Show: Blocks, Styles, Settings
- Enable: Compact Mode
- Result: Focused, minimal interface

**For Detail-Oriented Users**
- Show: All panels
- Enable: Full width right panel
- Disable: Compact Mode
- Result: Complete control over every detail

**For Mobile Users**
- Hide: Layers, History
- Compact Mode: On
- Panel Width: 150px (minimal)
- Result: Maximum canvas space on small screens

### Tips
- Your customization is saved locally (not in the cloud)
- Each browser/computer can have different settings
- You can close the settings panel by clicking the X or pressing Escape
- Changes take effect immediately

## üÜò Troubleshooting

### Tooltips Not Showing
- Check if you're hovering directly over the button
- Try using keyboard (Tab to focus, then wait for tooltip)
- Make sure tooltips aren't hidden by other panels

### History Not Updating
- History updates automatically after changes
- Check that the panel is visible (use settings to show it)
- Reload the page to refresh history

### Customization Not Saving
- Click "Save Changes" button (not just closing the panel)
- Check your browser's localStorage isn't disabled
- Try resetting to defaults and customizing again

### Can't Find Settings Button
- Look in the bottom-right corner for ‚öôÔ∏è
- Make sure it's not hidden behind another window
- Try scrolling or moving other panels
